# RFWG 报告结构与写作规范

> 供 AI 在「综合成稿」阶段查阅。目标：产出对**人类**和**AI**都友好的报告（Markdown 分件 + 单文件 HTML）。

## Markdown 产物（过程稿，边做边落盘）

| 文件 | 内容 | 生成方式 |
|------|------|----------|
| `01-完整消息.md` | 全量清洗时间线 + 概览统计 | `build_report_md.py` |
| `02-<主题>-精华.md` | 主题命中 ±N 上下文合并片段 | `build_report_md.py --topic` |
| `03-<对象>-发言精华.md` | 重点人物月度观点（人工/AI 提炼自 people/） | 读 `build_people_md.py` 产出后提炼 |
| `04-图片信息提取.md` | 图片判读结论 + 有价值清单 | 看 sheets 后写 + `sort_images.py` |
| `05-<对象>-朋友圈.md` | 朋友圈补充（若涉及某人） | 读 `decrypt_moments.py` 产出后提炼 |
| `images/`、`people/`、`*.json` | 原始/中间产物，供溯源 | 各脚本 |

## HTML 终稿（面向阅读）

用 `assets/report-template.html` 为骨架。**推荐 section 顺序**（按需增减）：

1. **执行摘要 / TL;DR**：一段话讲清主题结论 + 3~5 条要点 + 关键提醒。放最前，AI 可直接摘。
2. **方法与数据来源**：数据规模、时间范围、采集管线（配管线 SVG）、诚实边界（取不到什么）。
3. **核心概念 / 主题拆解**：定义、对比表、能力清单——**逻辑关系一律做 SVG**。
4. **关键人物 / 时间线**：重点对象的观点与实操时间线（配时间线 SVG）。
5. **多方观点 / 分歧**：立场光谱或对比（配 SVG）。
6. **交叉印证**：多来源（群聊/朋友圈/图片）对齐（配韦恩图 SVG + 对照表）。
7. **信号 vs 噪音**：明确哪些值得读、哪些是噪音；给出未解的真问题。
8. **附录**：人物对照、术语表、数据产物清单。

## SVG 图型速查（都用纯 `<text>`，viewBox 自适应）

- **管线图**：`<rect rx>` 节点 + `<path marker-end>` 箭头（定义 `<marker>`）。
- **对比表**：两列色块（中性色 vs 主题色），底部一行结论条。
- **能力卡片网格**：`<rect>` 卡 + 编号圆 `<circle>` + 标题/说明 `<text>`。
- **时间线**：竖轴 `<line>` + 圆点 `<circle>` + 右侧圆角条，日期用强调色。
- **立场光谱**：渐变 `<rect>` 长条 + 节点圆 + 说明卡。
- **韦恩图**：3 个半透明 `<circle fill-opacity>` 交叠 + 圆心标题。

## 硬性质量要求

- 事实性结论**必须带来源 + 时间戳**（如「群友A 07-16 12:21 朋友圈（示例）」）。
- **对外分发的报告默认匿名化**：真实姓名/wxid/实名雇主等可识别信息仅保留在**本地私稿**；对外版用「群友A/角色X」等占位，避免公开第三方个人信息。
- 区分**信号与噪音**，不堆砌；玩梗/情绪流归为噪音并说明。
- 外部人物观点如实归属，不美化、不断章取义。
- 生成后**必须浏览器验收**（见 SKILL.md）：无横向溢出、SVG 不越界、TOC 高亮正常、各 section 可达。
- 中文为主；术语首次出现给中英对照。
